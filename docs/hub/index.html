🧠 Ollama — IA Local

Toda leitura de PDF, análise de conformidade técnica e extração de dados não-estruturados é delegada ao Ollama rodando na GPU local (AMD RX 7600).

Modelos disponíveis

  • llama3.1:8b — análise de documentos PT-BR
  • qwen2.5:7b — alternativa geral
  • qwen2.5-coder:1.5b-base — código rápido
  • nomic-embed-text — embeddings/busca semântica

🐍 Python — Ambientes Isolados

Regra de Ouro: Nunca instale pacotes no Python global. Todo projeto tem seu próprio .venv.

# Criar e ativar
python3 -m venv .venv
source .venv/bin/activate

# Instalar dependências
pip install -r requirements.txt

🐍 Regras de Desenvolvimento Python

  1. Ambientes Virtuais Obrigatórios: NUNCA instale pacotes no Python global (pip install package). Sempre crie e utilize um ambiente virtual na pasta do projeto:
  2. Criar: python3 -m venv .venv
  3. Ativar: source .venv/bin/activate
  4. Registrar: Adicionar .venv ao .gitignore.

  5. Dependências: Mantenha sempre um requirements.txt atualizado na raiz de cada projeto para facilitar o setup via Ollama/Agente.

  6. Arquivos de Protocolo (ZIP): NUNCA suba arquivos .zip de protocolos da CEMIG ou documentos brutos de clientes no Git.

  7. O Git deve conter apenas: Código, Markdown, JSON de dados e PDFs de normas.
  8. Arquivos de envio (Zips de documentos) devem ser salvos APENAS na tarefa do Redmine correspondente.

⚡ Repositório: Cemig

Workflow completo de protocolos de ligação nova e aumento de carga.

Estrutura de Pastas

📁 clientes/

Pasta por cliente. Cada um tem subpastas vN/ para cada versão do protocolo enviado.

📁 normas/

Biblioteca de normas CEMIG (PDF + texto extraído). Fonte da verdade para validação técnica.

📁 docs/

Fluxo de trabalho, checklists e guias de avaliação de clientes.

📁 scripts/

Validador de normas ND (validador_nd.py) e ferramentas de apoio técnico.

Workflow de Submissão — Passo a Passo
  • 1
    Triagem de Titularidade (Fase Zero)
    Valide a conta de energia (fatura atualizada). O nome na fatura coincide com o solicitante? Se não, o primeiro protocolo DEVE ser a "Troca de Titularidade".
  • 2
    Definição Técnica (ND 5.1 / 5.2)
    Execute o validador para definir materiais: python3 scripts/validador_nd.py [5.1 ou 5.2] [carga]
    Confirme que disjuntor e cabos batem com o projeto. Para disjuntores >200A, verifique a marca na lista PEC-11.
  • 3
    Checklist de Documentos
    Pessoa Física: RG/CPF ou CNH + documento do imóvel + procuração com poderes específicos para "Cemig".
    Pessoa Jurídica: Contrato Social/CCMEI + Cartão CNPJ + RG/CPF do sócio administrador + procuração.
  • 4
    Protocolo no APR Web
    Os dados digitados no portal devem ser idênticos ao Formulário Excel e ao Projeto Técnico. Salve o comprovante imediatamente em vN/protocolo/.
  • 5
    Gestão de Rejeição (Retorno)
    Copie o motivo da rejeição para vN/retorno/motivo_rejeicao.txt. Se erro técnico, rode o validador_nd.py novamente para encontrar a divergência.
Documentos Obrigatórios — BT (Baixa Tensão)
DocumentoPessoa FísicaPessoa Jurídica
CPF + Documento com foto✅✅ (sócio administrador)
Contrato Social / CCMEI—✅
Cartão CNPJ—✅
Documento do imóvel (IPTU, Escritura, Habite-se, etc.)✅✅
Procuração com poderes específicos para "Cemig"Quando aplicável✅

🔴 Repositório: Redmine Automation

Automação completa do Redmine via API REST — tarefas, uploads e wikis.

🔑
Configure o arquivo .env antes de qualquer script: REDMINE_URL, REDMINE_API_KEY, EMAIL_USER, EMAIL_PASS, PROTHEUS_DB_PASS. Execute source .env antes de rodar.

📁 scripts/

Scripts de automação Python. Principal: enriquecer_tarefas.py para enriquecer issues com dados do cliente.

python scripts/enriquecer_tarefas.py

📁 docs/

Guias de renderização HTML e instruções da Wiki. Referência: docs/instrucoes-redmine-wiki.md.

📁 templates/

Templates HTML/CSS para Wikis ricas no Redmine com glassmorphism e dark mode.

Como Subir HTML Rico na Wiki
  • 1
    Suba o arquivo .html nos Arquivos do projeto no Redmine.
  • 2
    Anote o ID do anexo gerado após o upload.
  • 3
    Use o plugin View Customize para injetar um script jQuery que carregue o conteúdo no srcdoc de um iframe na wiki.
